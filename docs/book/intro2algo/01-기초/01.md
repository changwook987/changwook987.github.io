## 연습문제 1

1.  "정렬" 또는 "볼록 덮개 찾기" 계산 문제 중 하나를 골라 이런 문제가 발생하는 현실의 예를 제시하라.
    ::: details 보기
    - 정렬
        1. 물건을 가격이 저렴한 순서대로 보고 싶을 때
        2. 반의 자리를 정할 때 키가 작은 순서대로 앞 자리에 앉아야 할 때
    - 볼록 덮개 찾기
        1. 여러 크기의 사진을 모두 담을 수 있는 크기의 상자를 고를 때
        2. :robot: 동물들 전체를 **가장 적은 길이의 울타리**로, 즉 **가장 팽팽하게** 둘러싸는 방법 구하기. *by gpt*
    :::
2.  현실에서 속도 외에 효율성을 평가할 만한 다른 척도로 무엇이 있는지 제시하라.
    ::: details 보기
    - 공간 - 다음 두 코드가 있다.
        ```py
        def solution1(n: int):
            fibo = [0, 1]
            for _ in range(n - 1):
                fibo.append(fibo[-1] + fibo[-2])
            return fibo[n]

        def solution2(n: int):
            a, b = 0, 1
            for _ in range(n - 1):
                a, b = b, a + b
            return a
        ```
        두 코드 모두 $n$ 번째 피보나치 수를 계산하는 알고리즘이다. `solution1`은 $n$ 번째 까지의 모든 피보나치 수를 리스트에 저장한다. `solution2`는 $n$ 에 상관없이 마지막 2개의 피보나치수를 저장한다. 리스트에 숫자를 저장하는 시간을 계산하지 않을 때, 총 반복하는 횟수는 $n - 1$ 로 같지만 사용하는 메모리(*공간*)는 `solution2`가 `solution1`보다 효율적이라고 할 수 있다.
    :::
3.  예전에 본 적이 있는 자료구조 중 하나를 골라 그것의 장점과 한계를 각각 논하라.
    ::: details 보기
    - Linked List 
        - 아이템을 저장하는 노드가 앞, 뒤의 주소를 가지고있는 방식으로 리스트를 구현하는 자료구조이다. 이 때문에 중간에 있는 아이템을 삭제하거나 추가하는 연산의 비용이 굉장히 저렴하지만, $N$ 번째 아이템을 구하기 위해서는 저장하고 있는 가장 첫 노드에서 $N$ 번째 까지 반복해서 찾아가야하기 때문에 그런 연산의 비용이 비싸다는 단점이 있다.
    :::
4.  앞서 설명한 최단 경로 문제와 순회 판매원 문제는 어떻게 비슷한가. 그리고 어떻게 다른가?
    ::: details 보기
    최단 경로 문제외 순회 판매원 문제는 모두 최소 비용을 구하는 문제라는 점에서 비슷하다고 볼 수 있다. 하지만 최단 경로 문제는 가장 비용이 낮은 경로를 선택하는 방식으로 풀이하면 쉽게 풀리지만, 순회 판매원 문제는 당장은 최선이 아니더라도 최선에 포함되는 경로 때문에 가지치기를 하기 어려워진다.
    :::
5.  현실의 문제 중 최적해만 의미가 있는 문제를 제시하라. 반대로 최적은 아니지만 "근접한" 해를 구해도 충분한 문제를 제시하라.
    ::: details 보기
    - 최적해
        > 정확도를 요구하는 문제
        1. 우주선 시뮬레이션
        2. 은행/금융
        3. 암호/보안
    - 근접한 해
        > 현실적으로 최적해를 계산하는 비용이 비싼 문제.
        1. 자동차 네비게이션
    :::
